# 文章页右侧悬浮目录轨道设计

## 状态

设计已确认并完成实现可行性审查。本文档按用户要求写入仓库但不提交；实现阶段不创建分支，不引入 TDD 流程。

## 目标

在文章页右侧增加一个紧凑的悬浮目录轨道，借鉴参考站的“轨道/详情”切换方式，同时保持当前博客的极简黑白视觉、Swup 页面过渡和低运行成本。

目录用于阅读位置查看，不承担文章导航，不提供点击跳转。

## 已确认的交互

### 收起态

- 仅在宽屏桌面显示：视口宽度大于 `1280px`。
- 右侧显示 H1、H2、H3 的水平虚线横杠。
- H1、H2、H3 通过横杠长度、缩进和透明度形成递进关系。
- 当前阅读标题对应的横杠轻微放大、提亮，并播放一次低幅波浪过渡。
- 不显示文字标题、摘要、图标或操作按钮。

### 展开态

- 鼠标进入右侧轨道热区后展开；鼠标离开约 `250ms` 后收起。
- 键盘 `focus-visible` 也可以展开，但不增加点击行为。
- 轨道固定在视口右侧，详情向左展开，不引起正文重排。
- 只显示当前阅读位置所属的 H1、H2、H3 路径，不列出全文标题。
- 每个标题只显示一行，过长文本使用省略号。
- 当前所在层级高亮，父级标题弱化；缺少某一级时直接省略。
- 标题组底部显示水平虚线进度轨、进度段和百分比。

### 非目标

- 不在轨道上实现锚点点击或文章跳转。
- 不使用持续循环的波浪动画。
- 不新增滚动库、GSAP 动画或独立的重型状态管理。
- 不替换移动端现有 MobileDock 的目录入口。

## 组件与数据流

新增独立的 `ArticleOutlineRail.astro` 外壳，放在 `#swup-container` 内，仅文章页渲染。组件只负责标记，不在可替换容器内注册全局生命周期监听器。

配套轻量 Controller，职责如下：

1. 在初始化阶段扫描 `.custom-md h1, h2, h3`。
2. 克隆标题后移除 `script`、`style`、`.anchor`、`.anchor-icon` 和 `[data-pagefind-ignore]`，再提取纯文本；空文本使用可用的 `aria-label` 或 ID 回退。
3. 按文档顺序生成标题模型，记录层级、ID、纯文本、元素引用和所属父级。
4. 为所有标题生成收起态横杠。
5. 根据当前阅读位置生成最多三行的 H1/H2/H3 路径。
6. 维护当前活动标题、进度百分比和 CSS 状态变量。
7. 在文章没有有效标题时隐藏整个组件。

横杠在初始化时生成一次；详情区固定保留三个标题行和一个进度区，只在活动标题变化时更新文本。收展只切换状态类和 CSS 变量，不在鼠标移动时重建节点。

现有 `TOCManager` 继续服务 Sidebar/Dock；右侧轨道不扩展它的模式，避免影响既有调用方。

## 替换范围与文件归属

新轨道替换当前右侧 `article-toc-wrapper`，不能与旧目录并存：

- `MainGridLayout.astro`：用 `ArticleOutlineRail` 替换旧右侧 TOC 标记，删除该布局中旧 `TOCManager`、点击监听、对齐逻辑、ResizeObserver 和 MutationObserver。
- `src/styles/layout/grid.css`：删除旧 `.article-toc-*` 规则，保留主网格和 Swup 容器规则。
- `src/styles/toc.css`：继续供 Sidebar/Dock 使用，不随旧右侧目录删除。
- `src/styles/components/article-outline-rail.css`：承载新组件全部样式，并由 `src/styles/main.css` 统一导入。
- `src/utils/article-outline-controller.ts`：承载标题模型、位置缓存、进度与清理逻辑。
- 持久化 `Layout.astro`：只负责启动 Runtime；Runtime 的实现可独立放入工具模块，避免把业务逻辑堆入布局。

## 视觉与动效

- 外层使用固定约 `18rem` 的右对齐区域，不动画宽度；收起态只有最右侧约 `3rem` 的轨道可以接收指针事件。
- 收起时详情层为透明、不可点击且不形成 hover 热区；轨道 hover 或 focus 后详情层才获得指针事件，防止鼠标靠近正文时误展开。
- 轨道本身不使用实体卡片背景。
- H1 横杠最长、H2 次之、H3 最短；每级有固定缩进。
- 横杠使用主题变量控制颜色，避免硬编码色值。
- 活动横杠使用 `transform` 放大约 `1.1` 倍，并提高对比度。
- 横杠栈内只放置一个共享的内联 SVG 波浪覆盖层。活动标题变化时更新其纵向位置、缩进和宽度，并播放一次低幅淡入/伸展；不为每个标题创建 SVG，也不动画 path 数据。
- 展开过渡控制在 `240–280ms`，只动画 `transform` 和 `opacity`。
- 收起延迟约 `250ms`，避免鼠标从轨道移动到详情区域时闪烁。
- 详情标题按 H1/H2/H3 缩进，当前标题高亮，父级降低透明度。
- 进度轨使用一层稳定的虚线基线和一层 `scaleX()` 进度段。
- `prefers-reduced-motion: reduce` 时移除位移、波浪和延迟，仅保留快速透明度切换。

## 滚动进度与活动标题

- 固定阅读偏移为 `80px`。正文绝对顶部减去偏移得到起点；正文绝对底部减去视口高度再加偏移得到终点。
- 进度按 `(scrollY - start) / max(1, end - start)` 计算并限制在 `0–100`。正文短于有效视口时，在正文进入阅读线后直接视为完成，避免除零和负区间。
- 活动标题取 `scrollY + 80px` 之前最近的 H1/H2/H3；阅读线尚未经过首个标题时使用第一个标题，经过末尾后保持最后一个标题。
- 当前标题索引由缓存的位置数组通过二分查找得到，不在滚动帧内遍历 DOM。
- 滚动监听使用 `{ passive: true }`，通过单个 `requestAnimationFrame` 合并更新。
- 标题活动索引不变时，只更新进度 CSS 变量和整数百分比文本。
- 标题位置只在初始化、窗口尺寸变化、正文高度变化或密码解密后重新测量。
- 使用一个 `ResizeObserver` 处理图片、字体和折叠内容导致的正文高度变化。

## Swup 生命周期与清理

启动器放在持久化的 `Layout.astro`，位于三个 Swup 容器之外。使用 `window.__articleOutlineRailRuntime` 作为单例保护，Runtime 自身的事件监听通过一个长期 `AbortController` 只注册一次，并持有当前页面的 Controller：

- 首次完整加载：若当前 DOM 已就绪则立即初始化，避免错过首次 `astro:page-load`。
- `astro:before-swap`：只销毁当前页面 Controller，取消 RAF，移除页面监听器，断开 `ResizeObserver`；Runtime 的全局监听继续保留。
- `astro:page-load`：读取新文章 DOM 并初始化。
- `password:decrypted`：重新采集标题和位置。
- 重复初始化前先执行销毁，禁止多个 Controller 并存。
- 不使用未保存的 `100/200ms` 延时任务，避免快速导航时旧任务污染新页面。

因为右侧轨道外壳位于 Swup 容器内而 Runtime 位于容器外，Swup 替换后 Controller 只重新绑定新节点，不会重复注册文档级事件。

现有 Dock 作为窄屏回退入口，本次同步收口其 `TOCManager` 所有权：只保留一个实例、补齐 `indicatorId`，页面切换时清理，避免重复打开目录累积 `IntersectionObserver`。

## 响应式与可访问性

- `>1280px` 且设备满足 `(hover: hover) and (pointer: fine)`：显示右侧轨道，详情向左展开。
- `<=1280px`：隐藏右侧轨道，继续使用现有 Dock。
- `<=768px`：由 MobileDock 提供目录入口。
- 大尺寸粗指针设备同样隐藏轨道并使用 Dock，不留下无法展开的横杠。
- 轨道元素设置 `tabindex="0"` 和本地化的可访问名称；`:focus-visible` / `:focus-within` 与 hover 使用同一展开状态，键盘用户可以查看详情。
- `1281–1663px` 的外侧留白不足以容纳完整详情，展开层允许暂时覆盖正文右缘，并使用无边框、无圆角、无阴影的 `var(--page-bg)` 平面底色保证标题可读；收起态不覆盖正文。
- `>=1664px` 时详情完整位于正文外侧留白，可保持透明背景。
- 详情文本提供完整 `title`，但不使用高频 `aria-live`，避免滚动时连续播报。
- 颜色、边框、间距和圆角使用现有主题变量，兼容 light/dark。
- 不产生横向滚动；中等桌面宽度仅在用户主动 hover/focus 查看详情时短暂覆盖正文右缘。

## 方案权衡

### 采用：独立轨道组件 + 独立 Controller

改动集中、生命周期边界清楚，能够保留 Dock/Sidebar 的既有行为；滚动状态只更新必要的 CSS 属性，运行成本最低。

### 未采用：扩展 `TOCManager`

现有类同时被 Sidebar 和 Dock 使用，已有实例清理与多选活动状态问题。把轨道模式塞入同一类会扩大回归范围。

### 未采用：Svelte 岛组件

虽然状态表达方便，但会增加 hydration 和 Swup 重挂载成本，对这个只包含少量 DOM 状态的组件没有足够收益。

## 验收标准

1. 在 1281px、1440px、1664px、1920px 下检查轨道：详情均向左展开且不被裁切；1664px 以下覆盖层文字清晰、移出后正文立即恢复无遮挡。
2. 在 1280px 以下轨道隐藏，Dock 目录仍可用；移动端不与 MobileDock 重复。
3. 收起态能看到 H1/H2/H3 递进横杠，展开态能看到当前 H1/H2/H3 路径。
4. 当前标题高亮、横杠有一次轻微放大和波浪过渡，标题始终单行省略。
5. 顶部、中段、底部进度分别接近 `0%`、`50%`、`100%`，H3 活动时父级路径正确。
6. 图片延迟加载、密码解密和正文高度变化后，标题位置仍准确。
7. 连续进行文章间 Swup 切换，Observer、RAF 和事件监听器不会累积。
8. light/dark、`prefers-reduced-motion` 和键盘 focus-visible 状态可用。
9. 执行 `pnpm check`、`pnpm type-check`、`pnpm build`，并用浏览器完成桌面/窄屏手动回归。
10. 依次执行“文章 -> 文章 -> 非文章页 -> 文章”的 Swup 导航循环，确认全局 Runtime 只有一个、页面 Controller 每次正确替换。
