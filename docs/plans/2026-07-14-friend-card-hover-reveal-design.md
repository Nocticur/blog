# 友链卡片悬停揭示设计

## 背景

友链卡片的简介框当前在悬停时从 `2.2rem` 增高至 `7.5rem`。简介框处于正常文档流中，因此 CSS Grid 会重新计算所在行高度并推动后续行，造成同行卡片之间出现不自然的大块空白。

## 目标

- 卡片和网格行在悬停前后保持相同高度。
- 简介从卡片底部向上揭开，最终覆盖头像和名称。
- 保留纸张上移、旋转、阴影、圆角与胶带变化等现有动效。
- 支持键盘焦点、触屏设备与减少动画偏好。

## 交互设计

卡片维持固定的折叠态高度。简介层脱离正常文档流，绝对定位在卡片内容区域底部，并始终拥有展开态尺寸。默认状态通过向下位移只露出一行简介；桌面端悬停或键盘聚焦时，简介层向上移动到展开位置，覆盖头像和名称。

头像与名称在简介揭示时轻微上移并淡出，避免内容穿过简介边框。顶部类型胶带保持最高层级，不被简介覆盖。完整简介限制显示行数，超出部分省略。

## 组件与样式

- `FriendCard.astro`：保留链接、头像、名称和简介的现有语义结构；仅在层级控制确有需要时增加内容容器。
- `friend-card.css`：负责固定高度、绝对定位、层级与状态动画。
- `friends.css`：网格列数、间距和分页逻辑保持不变。

该交互不引入 JavaScript 状态。鼠标使用 `:hover`，键盘使用 `:focus-visible`；触屏设备保持折叠态，点击仍直接访问友链。

## 边界处理

- 长简介采用多行截断，不改变卡片尺寸。
- 长名称继续使用单行省略。
- `prefers-reduced-motion: reduce` 下取消位移过渡，直接显示最终状态。
- 仅在支持精确悬停的设备上启用鼠标悬停揭示，避免移动端粘滞 hover。

## 验证

- 对比任一卡片悬停前后的 Grid 行位置和页面总高度，数值应保持不变。
- 检查第一、第二、第三列及最后一行卡片，确认简介不会溢出纸张边界。
- 检查长名称、长简介、头像失败、亮色和暗色主题。
- 检查键盘焦点、触屏宽度和减少动画模式。
