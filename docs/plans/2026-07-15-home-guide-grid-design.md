# 首页站点导览九宫格设计

## 目标

将首页数据层中“社交联系”和“技能图标”两条跑马灯替换为紧凑的站点导览九宫格。该区域服务于访客阅读和探索，而不是展示个人技术栈。

## 布局

保留现有的黑色描边、低圆角、白底与低透明度角色插画。标题从“能力矩阵”调整为“站点导览”。桌面端使用三列九宫格，移动端改为单列卡片。

1. 从这里开始：指向最新文章列表。
2. 文章归档：显示文章、分类、标签统计并进入归档页。
3. 随机漫游：指向随机文章。
4. 置顶文章：跨两列，轮换展示所有置顶文章。
5. 最新更新：轮换展示最近发布的文章。
6. 分类索引：展示前三个分类并进入分类页。
7. 标签索引：展示前三个标签并进入标签页。
8. 订阅与联系：默认展示 QQ、GitHub、RSS；键盘聚焦或鼠标悬停后展开全部链接。

## 数据与交互

文章直接使用现有 `getSortedPosts()` 结果。置顶轮换仅使用 `data.pinned` 项；最新轮换过滤置顶项后取最近文章，数据不足时安全回退。文章卡每 5 秒轮换，悬停或聚焦时暂停，并提供可点击的分页圆点。

所有卡片保留 160-240ms 的 `transform` 与颜色微交互，避免布局偏移。`prefers-reduced-motion` 下禁用自动轮换及动画。联系方式卡在触摸设备上通过点击切换展开状态，支持 Escape 收起。

## 验证

为九宫格的数据构建工具补充单元测试，覆盖置顶/最新文章筛选及随机文章链接。随后执行类型检查、单元测试和生产构建，并在桌面与移动视口检查布局、轮换及联系卡交互。
